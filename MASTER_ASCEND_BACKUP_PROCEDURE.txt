MASTER ASCEND BACKUP PROCEDURE
==============================

Goal
----
Back up Ascend project files manually from the Ascend workspace into:

C:\Users\ISBE\ascend-pipelines

Then upload them to GitHub:

https://github.com/changshuren/ascend-pipelines


OVERVIEW
--------
You will do 4 things:

1. Create a local backup folder
2. Copy files from Ascend into matching local folders
3. Include safe CSV datasets if needed
4. Commit and push everything to GitHub


PART 1 — CREATE THE LOCAL BACKUP FOLDER
---------------------------------------
Open Command Prompt and run:

mkdir C:\Users\ISBE\ascend-pipelines
cd C:\Users\ISBE\ascend-pipelines
git init


PART 2 — CREATE MAIN FOLDERS
----------------------------
Run:

mkdir C:\Users\ISBE\ascend-pipelines\flows
mkdir C:\Users\ISBE\ascend-pipelines\data
mkdir C:\Users\ISBE\ascend-pipelines\connections
mkdir C:\Users\ISBE\ascend-pipelines\automations
mkdir C:\Users\ISBE\ascend-pipelines\macros
mkdir C:\Users\ISBE\ascend-pipelines\src

Optional if needed later:

mkdir C:\Users\ISBE\ascend-pipelines\profiles
mkdir C:\Users\ISBE\ascend-pipelines\otto


PART 3 — COPY ROOT FILES FIRST
------------------------------
Copy these first from Ascend:

1. README.md
   Save as:
   C:\Users\ISBE\ascend-pipelines\README.md

2. ascend_project.yaml
   Save as:
   C:\Users\ISBE\ascend-pipelines\ascend_project.yaml

Optional:
3. .gitignore
4. .ascendignore


PART 4 — HOW TO COPY EACH FILE
------------------------------
For each file in Ascend:

1. Open the file in Ascend editor
2. Click inside the editor
3. Press Ctrl + A
4. Press Ctrl + C

Then on your computer:

1. Open Notepad or VS Code
2. Press Ctrl + V
3. Save the file in the matching local path
4. Use the exact same file name

PART 6 — SAFE DATA RULES
------------------------
Include CSV files only if they are safe to publish.

Safe to publish:
- sample/demo data
- fake names
- example.com emails
- non-sensitive content

Do NOT publish:
- real personal data
- private customer emails
- passwords
- tokens
- secrets

If needed, replace real data with sample data before upload.


PART 7 — OPTIONAL OTHER FOLDERS TO BACK UP
------------------------------------------

2. connections/   (review first for secrets)
3. macros/

PART 8 — VERIFY LOCAL FILES
---------------------------
Run:

dir /s C:\Users\ISBE\ascend-pipelines

This shows all files currently saved.


PART 9 — UPLOAD EVERYTHING TO GITHUB
------------------------------------
Open Command Prompt and run:

cd C:\Users\ISBE\ascend-pipelines
git status
git add .
git commit -m "Add Ascend project backups"
git branch -M main
git remote set-url origin https://github.com/changshuren/ascend-pipelines.git
git push

If origin was never added before, run instead:

git remote add origin https://github.com/changshuren/ascend-pipelines.git

Then:

git push -u origin main

COMMON ERRORS
-------------
1. "remote origin already exists"
   Use:
   git remote set-url origin https://github.com/changshuren/ascend-pipelines.git

2. "Please tell me who you are"
   Use:
   git config --global user.name "changshuren"
   git config --global user.email "changshuren@gmail.com"

3. "failed to push some refs"
   Repo may already contain content. Ask Otto for the next fix.

4. Authentication popup/browser sign-in
   Complete sign-in, then run:
   git push -u origin main
